Philosophie git

  • Dépôt distribué
  • Philosophie axée sur les branches
  • Code quality (reviewing)
  • Participatif (pull request, fork)
  • Rapidité d'opération

Avantages

  • Philosophie de branches
  • Rapidité d’exécution (exécution local)
  • Favorise la qualité du code (Code review integré : pull request)
  • Commit partiel (split de commit pour un même fichier)
  • Repo local (offline work, branche temporaire local,...)
  • Travaux concurrents simple (git stash ou commit local)
  • Permet le partage exclusif (Dev to dev sans passer par l'origin)
  • Merge efficace (meilleurs gestion de conflits)
  • Pas de Tree Conflict (SVN)
