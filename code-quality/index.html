Les conventions de nommage

Les conventions de nommage permettent d'identifier le type d'un élément au premier coup d'oeil.

  • Prefix
  • Casing

Convention de nommage

Exemple :

_private // variable privée avec préfix '_'
aParam   // préfix a pour les paramètres (arguments)
onEvent  // préfix 'on' lors d'évènements
IMyInterface // Interface préfixée avec un I majuscule

Casse

Exemple :

CONSTANTES    // UPPER_CASE
Class	      // Pascal case
methodsName   // camelCase
attribut_name // snake_case

Structure de fichier

  • Garder le même ordre de déclaration
    1. Private variables
    2. Constructeurs
    3. Propriétés Getter/Setter
    4. Methods
  • Découpler au maximum et splitter en fichier (class, sub class, behaviors, ...)
  • Architecture (design pattern, POO, ...)

Détecter le code a problème

  • Des méthodes trop longues
  • Des classes trop grandes
  • Trop de paramètres
  • Difficulté à lire le code
  • Multiple niveau d imbrication/indentation
  • Des objets sans méthodes (hors getter/setter ->POO)
  • Trop de méthode public static
  • Des tests trop compliqués
  • Si y a un doute, c est mauvais
